# Получение матрицы зависимостей

## CLI


### Основное применение

`uv run eol-matrix [matrix|check]`

Доступные команды

- `matrix` - Построение матрицы зависимостей

Разбито на подкоманды:

#### matrix

Строим матрицу зависимостей от версии языка и либ.

---

## Поведение

### Загрузка версий

1. Запрашиваем список версий из `https://endoflife.date/api/<lang>`
   - По умолчанию используем только актуальные версии (фильтр `isEOL: false`)
   - Флаг `--with-eol` включает все версии, включая EOL
2. Запрашиваем информацию о зависимостях через API пакетного менеджера (PyPI для Python, npm registry для Node и т.д.)
3. Строим матрицу совместимости на основе полей типа `requires_python`

### Фильтрация версий в аргументах

Аргументы могут содержать версионные операторы:

```bash
$ matrix python>=3.11 -l django>=4.1
```

- Версионный фильтр применяется как **постфильтр результатов**
- Если API поддерживает фильтрацию на уровне запроса — используем его

### Обработка ошибок

- **Критические ошибки** (не удалось получить список версий из `endoflife.date`, неверный рантайм): завершить работу с кодом 1 и сообщением об ошибке
- **Предупреждения для отдельных библиотек** — если для одной библиотеки данные недоступны, вывести предупреждение и продолжить
- Несуществующие комбинации версий приводят к пустому списку

## Вывод

### JSON-формат (`--format json`, по умолчанию)

```json
{
  "django52": [
    "3.10",
    "3.11",
    "3.12",
    "3.13",
    "3.14"
  ],
  "django60": [
    "3.12",
    "3.13",
    "3.14"
  ]
}
```

### Табличный формат (`--format table`)

```
Library   Python Versions
───────── ───────────────
django52  3.10, 3.11, 3.12, 3.13, 3.14
django60  3.12, 3.13, 3.14
```

## CLI

### Команда `matrix`

```bash
$ matrix <lang> -l <library> [--lib <library>...] [--with-eol] [--format json|table]
```

**Аргументы**

| Аргумент | Описание |
| --- | --- |
| `<lang>` | Имя рантайма (например `python`, `node`) |
| `-l, --lib` | Зависимость. Может быть указана несколько раз или через запятую. Формат: `name` или `name>=version` |
| `--with-eol` | Включить версии, закончившие поддержку |
| `--format` | Формат вывода: `json` (по умолчанию) или `table` |

**Примеры**

```bash
# Простая матрица
$ matrix python -l django

# Несколько зависимостей
$ matrix python -l django -l django-debug-toolbar

# С включением EOL-версий
$ matrix python -l django --with-eol

# Фильтрация по версии
$ matrix python>=3.11 -l django>=4.1

# Табличный формат
$ matrix python -l django --format table
```

**Пример вывода**


### Многомерная матрица

Текущий формат поддерживает одну зависимость. Нужно продумать вывод для нескольких зависимостей одновременно.

Возможные подходы:
- Вложенные ключи: `"django52": { "ddt20": [...] }`
- Массив комбинаций: `[ { "python": "3.12", "django": "5.2", "ddt": "2.0" }, ... ]`
- Плоский формат: строка на каждую пару (рантайм, зависимость)

### Обработка недоступных данных

- Пакет не найден в реестре
- Отсутствие поля `requires_python` (или аналога)
- Ошибки API (таймаут, rate limit)
- Несуществующие комбинации версий — выдать ошибку или предупреждение вместо пустого списка

### Кеш и оффлайн-режим

- Сохранение результатов запросов к `endoflife.date` и пакетным реестрам
- Флаг `--offline` для работы с кешем
- Флаг `--no-cache` для отключения кеша

### Расширенные возможности

- Поддержка дополнительных реестров (Go, Rust и т.д.)
- Экспорт в другие форматы (CSV, YAML)
- Группировка версий (диапазоны) в табличном выводе
- Поддержка операторов сравнения (`<=`, `!=`, `~=`)
